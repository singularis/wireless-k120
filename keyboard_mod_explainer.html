HARDWARE LAYER (KEYBOARD)

1. Physical Keypress on the Logitech K120

When you press a key (e.g., the 'A' key), the mechanical dome matrix underneath the keycap closes an electrical circuit. The internal standard USB controller chip inside the K120 detects this contact and packages the event into a standard USB HID report containing the keyboard key scan code.

Internal Wiring Layout

To fit the components inside the keyboard casing, you cut the main K120 USB cable and solder the wires directly to the ESP32-S3 Zero microcontroller pins.

💡 Technical Tip: S3 USB-OTG

Unlike normal microcontrollers, the **ESP32-S3** contains physical hardware pins (GPIO 19 and 20) wired directly to a built-in USB-OTG PHY. This lets the chip act as a USB Host directly, reading raw keyboard data streams without needing an external USB host controller chip.

Core Firmware logic

The ESP32-S3 firmware runs a concurrent loop: reading the USB host packets from GPIO 19/20, translating the keys, and broadcasting them out via the Bluetooth stack.

// Standard configuration code for ESP32S3 USB Host to BLE translation
#include <usb/usb_host.h>
#include <BleKeyboard.h>

BleKeyboard bleKeyboard("Sleeper K120", "Custom DIY");

void setup() {
  Serial.begin(115200);
  bleKeyboard.begin(); // Start Bluetooth Low Energy HID Keyboard stack
  
  // Initialize standard USB Host driver on GPIO 19 (D-) and 20 (D+)
  usb_host_config_t host_config = { .skip_phy_setup = false };
  usb_host_install(&host_config);
}

void loop() {
  if (bleKeyboard.isConnected()) {
    // Read raw HID report packets from the USB Host controller
    usb_transfer_t* transfer = get_usb_keyboard_report();
    if (transfer) {
      // Translate scan codes and send directly over BLE
      bleKeyboard.write(translate_scan_code(transfer->data));
    }
  }
}
